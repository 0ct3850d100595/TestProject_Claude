## 1. テスト方針

### 1.1 テスト種別

| 種別 | 説明 |
|------|------|
| 機能テスト | 各機能が仕様通りに動作することを確認 |
| 権限テスト | ロール別のアクセス制御が正しく機能することを確認 |
| 境界値テスト | 入力値の上限・下限・境界付近の動作を確認 |
| 異常系テスト | 不正入力・権限外アクセス時のエラー処理を確認 |
| 画面遷移テスト | 各画面間の遷移が仕様通りに動作することを確認 |

### 1.2 テスト優先度

| 優先度 | 基準 |
|--------|------|
| 高 | 業務に直結する中核機能（日報CRUD・認証・権限制御） |
| 中 | マスタ管理・コメント機能 |
| 低 | UI表示・ソート・ページネーション |

### 1.3 合否判定基準

- 期待結果と実際の結果が完全に一致する場合を「合格（Pass）」とする
- 一部でも差異がある場合は「不合格（Fail）」とし、不具合票を起票する

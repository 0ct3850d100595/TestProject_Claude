## 2. 共通仕様

### 2.1 ロール定義

| ロール | 説明 | 権限概要 |
|--------|------|----------|
| sales（営業担当） | 通常の営業社員 | 自分の日報のみ作成・編集可能 |
| manager（上長） | 営業担当の管理者 | 部下の日報閲覧・コメント可能 |
| admin（管理者） | システム管理者 | 全機能利用可能、マスタ管理 |

### 2.2 共通レイアウト

- ヘッダー: ログインユーザー名・ロール表示、ログアウトボタン
- サイドナビ: ダッシュボード / 日報一覧 / 顧客マスタ / 社員マスタ（権限に応じて表示）
- フッター: システム名・バージョン

### 2.3 共通バリデーション

- 必須項目は `*` マークで表示
- エラーメッセージは該当フィールドの直下に赤字で表示
- 保存・送信前に確認ダイアログを表示する

### 2.4 日付フォーマット

- 表示: `YYYY年MM月DD日`
- 入力: カレンダーピッカー使用（手入力も可）

## 6. SCR-004 日報作成・編集画面

### 6.1 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-004 |
| 画面名 | 日報作成・編集画面 |
| URL（新規） | `/reports/new` |
| URL（編集） | `/reports/{id}/edit` |
| 利用者 | salesロール（自分の日報のみ） |

### 6.2 画面構成

```
┌──────────────────────────────────────────────────────────┐
│ 日報作成                                                  │
│                                                          │
│ 報告日 *  [2026/04/20          📅]                       │
│                                                          │
│ ── 訪問記録 ─────────────────────────────────────────── │
│                                                          │
│  #1  顧客名 * [株式会社○○         ▼]                    │
│       訪問内容 *                                          │
│       ┌────────────────────────────────────────────┐     │
│       │ 新製品の提案を実施。先方の担当者より前向きな │     │
│       │ 回答を得た。次回見積書を持参する予定。      │     │
│       └────────────────────────────────────────────┘     │
│                                                          │
│  #2  顧客名 * [△△商事           ▼]                    │
│       訪問内容 *                                          │
│       ┌────────────────────────────────────────────┐     │
│       │ 定期フォロー訪問。特に進展なし。            │     │
│       └────────────────────────────────────────────┘     │
│                                                          │
│  [+ 訪問先を追加する]  [− 削除]                          │
│                                                          │
│ ── 今日の課題・相談 ─────────────────────────────────── │
│  ┌──────────────────────────────────────────────────┐    │
│  │ 競合他社が値引き攻勢をかけており、□□社の案件が  │    │
│  │ 難航している。上長へ相談したい。                │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│ ── 明日やること ──────────────────────────────────────── │
│  ┌──────────────────────────────────────────────────┐    │
│  │ ・□□社への見積書を作成する                      │    │
│  │ ・◇◇株式会社へアポイントを取る                 │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  [保存する]  [キャンセル]                                │
└──────────────────────────────────────────────────────────┘
```

### 6.3 入力項目

| No | 項目名 | 入力形式 | 必須 | バリデーション |
|----|--------|----------|------|----------------|
| 1 | 報告日 | 日付ピッカー | ✓ | 未来日不可。同一ユーザー・同一日付の重複不可 |
| 2 | 顧客名（訪問記録） | セレクトボックス（顧客マスタから選択） | ✓ | 顧客マスタに存在すること |
| 3 | 訪問内容（訪問記録） | テキストエリア | ✓ | 最大1000文字 |
| 4 | 今日の課題・相談 | テキストエリア | - | 最大2000文字 |
| 5 | 明日やること | テキストエリア | - | 最大2000文字 |

### 6.4 訪問記録の動的追加・削除

- 初期表示: 訪問記録は1行表示
- 「+ 訪問先を追加する」ボタンで行を追加（上限なし）
- 2行以上ある場合は各行に「削除（−）」ボタンを表示
- 1行のみの場合は削除ボタンを非表示

### 6.5 ボタン・アクション

| ボタン名 | 処理内容 |
|----------|----------|
| 保存する | バリデーション後、日報をDBに保存。成功時はSCR-005（詳細）へ遷移 |
| キャンセル | 入力内容を破棄して前画面へ戻る（確認ダイアログあり） |
| + 訪問先を追加する | 訪問記録の入力行を1行追加 |
| − （削除） | 対象の訪問記録行を削除（確認ダイアログあり） |

### 6.6 エラーメッセージ

| 条件 | メッセージ |
|------|-----------|
| 同日付の日報が既存 | この日付の日報はすでに作成されています |
| 未来日を指定 | 未来の日付は指定できません |
| 顧客未選択 | 顧客を選択してください |
| 訪問内容が空 | 訪問内容を入力してください |
